# AM-102-05: Handle mid-stream errors

Story: [AM-102](../../stories/01-core-api/am-102.md)
Status: todo

---

## Description

When an error occurs during streaming, emit a chunk with finish_reason: "error" instead of dropping the connection.

## Acceptance Criteria

- [ ] HTTP status remains 200 (already started streaming)
- [ ] Error chunk has finish_reason: "error" in choices
- [ ] Error details included in the chunk (error field at top level)
- [ ] Stream terminates with data: [DONE] after error chunk
- [ ] Consumer can detect the error by checking finish_reason
