# AM-105-01: Implement fallback router

Story: [AM-105](../../stories/01-core-api/am-105.md)
Status: todo

---

## Description

When models[] + route: "fallback" is set, try models in order.

## Acceptance Criteria

- [ ] Iterates models array in order
- [ ] On provider error or timeout, advances to next model
- [ ] Returns response from first successful model
- [ ] Does not retry the same model (moves forward only)
- [ ] Handles mixed providers in the array (e.g., anthropic then openai)
