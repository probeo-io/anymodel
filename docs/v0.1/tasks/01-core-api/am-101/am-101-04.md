# AM-101-04: Validate request parameters

Story: [AM-101](../../stories/01-core-api/am-101.md)
Status: todo

---

## Description

Validate required fields (model, messages) and optional parameter types/ranges (temperature 0-2, top_p 0-1, etc). Return 400 errors for invalid input.

## Acceptance Criteria

- [ ] Returns 400 if model is missing or empty
- [ ] Returns 400 if messages is missing, empty, or not an array
- [ ] Returns 400 if temperature is outside 0.0–2.0
- [ ] Returns 400 if top_p is outside 0.0–1.0
- [ ] Returns 400 if top_logprobs is set without logprobs: true
- [ ] Returns 400 if stop has more than 4 sequences
- [ ] Error response matches standard format: { error: { code: 400, message, metadata } }
- [ ] Valid requests pass through without modification
