## Description

Estimate token counts for truncation decisions.

## Acceptance Criteria

- [ ] Provides rough token count for a string (character/4 heuristic or tiktoken)
- [ ] Works for all providers (provider-agnostic estimation)
- [ ] Fast enough for real-time use (< 1ms for typical message arrays)
- [ ] Errs on the side of overestimating (safer to truncate slightly more than needed)
