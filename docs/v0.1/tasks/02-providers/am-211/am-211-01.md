# AM-211-01: Implement retry logic with exponential backoff

Story: [AM-211](../../stories/02-providers/am-211.md)
Status: todo

---

## Description

Auto-retry transient errors.

## Acceptance Criteria

- [ ] Retries on 429 and 5xx responses
- [ ] Does not retry on 400, 401, 422 (client errors)
- [ ] Exponential backoff: delay = initialBackoff * 2^attempt + jitter
- [ ] Reads Retry-After header and uses it if longer than calculated backoff
- [ ] Max retries configurable (default: 3)
- [ ] Initial backoff configurable per provider (default: 1000ms)
