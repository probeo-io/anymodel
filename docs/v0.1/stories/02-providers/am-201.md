## AM-201: Provider adapter contract
**As a** contributor
**I want** a clear `ProviderAdapter` interface
**So that** I can add new providers by implementing a standard set of methods

**Acceptance Criteria:**
- Interface requires: `name`, `translateRequest()`, `translateResponse()`, `translateStream()`, `translateError()`, `listModels()`, `supportsParameter()`
- All adapters registered via a provider registry
- Adding a new provider requires no changes to the router or core — only a new adapter file
