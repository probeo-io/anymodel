## AM-215: Parameter support discovery
**As a** contributor
**I want** each adapter to report which parameters it supports
**So that** the router can strip unsupported params and power the `require_parameters` filter

**Acceptance Criteria:**
- `adapter.supportsParameter('tools')` returns boolean
- `supported_parameters` populated in model list endpoint
- Unsupported params stripped before forwarding (no provider errors from unknown params)
- `require_parameters` routing filter uses this to select compatible providers
