## AM-211: Automatic retry with backoff
**As a** developer
**I want** failed requests automatically retried with exponential backoff
**So that** transient errors don't require manual retry logic

**Acceptance Criteria:**
- 429 and 5xx responses trigger automatic retry
- Exponential backoff with jitter
- Reads `Retry-After` header when present
- Configurable max retries (default: 3) and initial backoff per provider
- When `models` fallback array is set, moves to next model instead of retrying same one
