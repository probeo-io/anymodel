## AM-210: Provider error mapping
**As a** developer
**I want** all provider errors mapped to a consistent format
**So that** error handling is the same regardless of which provider failed

**Acceptance Criteria:**
- 401/403 → 401 (Invalid API key)
- 429 → 429 (Rate limit)
- 400 → 400 (Invalid request)
- 500/502/503 → 502 (Provider unavailable)
- Timeout → 408 (Request timeout)
- Context too long → 400 (Invalid request)
- Original error preserved in `metadata.raw`
- `metadata.provider_name` identifies which provider errored
