## AM-213: Fallback model routing
**As a** developer
**I want** `models` + `route: "fallback"` to try models in order
**So that** my application is resilient to individual model/provider failures

**Acceptance Criteria:**
- On error or timeout, automatically tries next model in the array
- Response `model` reflects which model actually served
- Total latency includes all failed attempts
- Generation stats record all attempts (provider_responses array)
