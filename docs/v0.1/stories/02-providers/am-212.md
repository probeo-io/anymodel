## AM-212: Per-provider rate limit tracking
**As a** developer
**I want** anymodel to track rate limits per provider
**So that** it can avoid hitting limits and make smarter routing decisions

**Acceptance Criteria:**
- Reads `x-ratelimit-*` headers from provider responses
- Tracks remaining requests/tokens per provider
- Rate limit info available via `client.providers.rateLimits()`
