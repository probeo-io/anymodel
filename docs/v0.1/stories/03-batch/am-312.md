## AM-312: Batch cleanup and retention
**As a** developer
**I want** old batch data auto-cleaned after a configurable retention period
**So that** disk usage doesn't grow unbounded

**Acceptance Criteria:**
- `batch.retentionDays` in config (default: 30)
- Cleanup runs on client/server startup
- Only deletes batches in terminal status (completed/failed/cancelled)
- `client.batches.delete(id)` for manual cleanup
- Deletes the entire batch directory from disk
