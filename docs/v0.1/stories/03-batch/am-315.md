## AM-315: Concurrent fallback adapter
**As a** developer using providers without native batch (Google, custom)
**I want** batches to work via concurrent execution
**So that** the batch interface is consistent even without native support

**Acceptance Criteria:**
- Sends requests concurrently with configurable limit (`batch.concurrencyFallback`, default: 10)
- Each request goes through the standard chat completion path
- Results collected and written to `results.jsonl` as they complete
- `completed`/`failed` counters update in real-time
- `batch_mode: "concurrent"` in response
- Full price (no batch discount)
- Failed individual requests retried per retry config
