## AM-308: Resume polling a previous batch
**As a** developer
**I want to** resume polling a batch from a previous process via `client.batches.poll(id)`
**So that** I can reconnect to long-running batches after a restart

**Acceptance Criteria:**
- Reads `provider.json` from disk to get upstream batch ID
- Resumes polling upstream API
- `onProgress` callback works the same as `createAndPoll`
- Returns results when complete
- Works for batches created hours/days ago (within provider retention)
