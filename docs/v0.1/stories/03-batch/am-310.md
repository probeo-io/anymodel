## AM-310: Batch state persistence
**As a** developer
**I want** all batch state persisted to disk
**So that** batches survive process restarts and results aren't lost

**Acceptance Criteria:**
- Default dir: `~/.anymodel/batches/`
- Configurable via `ANYMODEL_BATCH_DIR` or config `batch.dir`
- Per-batch directory: `{id}/meta.json`, `requests.jsonl`, `results.jsonl`, `provider.json`
- `meta.json` updated on each status change
- `results.jsonl` appended incrementally as results arrive
- `provider.json` stores upstream provider batch IDs
