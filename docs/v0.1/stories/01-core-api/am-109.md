## AM-109: Provider/model naming
**As a** developer
**I want to** reference models as `provider/model`
**So that** I can unambiguously target any model on any provider

**Acceptance Criteria:**
- `anthropic/claude-sonnet-4-6`, `openai/gpt-4o`, `google/gemini-2.5-pro` all resolve correctly
- Custom providers: `groq/llama-3.3-70b`, `my-ollama/mistral` work via config
- Invalid provider or model returns 400 error with clear message
