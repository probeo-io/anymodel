## AM-112: Standardized error format
**As a** developer
**I want** all errors to follow a consistent format with `code`, `message`, and `metadata`
**So that** I can handle errors uniformly regardless of the underlying provider

**Acceptance Criteria:**
- Error shape: `{ error: { code, message, metadata: { provider_name, raw } } }`
- HTTP status matches `error.code`
- Provider-specific error details preserved in `metadata.raw`
- Codes: 400, 401, 408, 422, 429, 500, 502, 503
