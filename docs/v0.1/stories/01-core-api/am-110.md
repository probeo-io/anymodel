## AM-110: Model aliases
**As a** developer
**I want to** define aliases like `default`, `fast`, `smart`, `cheap`
**So that** I can switch models without changing application code

**Acceptance Criteria:**
- Aliases defined in config resolve to their target `provider/model`
- `client.chat.completions.create({ model: 'default', ... })` works
- Response `model` field shows the actual model used, not the alias
- Aliases can be overridden per-environment via config resolution order
