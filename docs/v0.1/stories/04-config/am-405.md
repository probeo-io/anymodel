## AM-405: Config resolution order
**As a** developer
**I want** a clear config precedence order
**So that** I know which setting wins when multiple sources conflict

**Acceptance Criteria:**
- Order (highest to lowest): programmatic options → local config → global config → env vars
- Env vars can override config file keys
- Merges deeply (provider-level, not top-level replacement)
