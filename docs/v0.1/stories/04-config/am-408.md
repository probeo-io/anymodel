## AM-408: Model aliases
**As a** developer
**I want** to define model aliases like `default`, `fast`, `smart`, `cheap`
**So that** I can switch models globally without changing application code

**Acceptance Criteria:**
- Defined in config under `aliases`
- Alias used as `model` in requests resolves to target `provider/model`
- Response `model` field shows the actual model, not the alias
- Built-in defaults: none (user must define their own)
- Aliases can reference other aliases (one level deep only, no cycles)
