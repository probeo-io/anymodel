## AM-401: Provider key env vars
**As a** developer
**I want** to configure provider API keys via standard environment variables
**So that** I can use anymodel without a config file

**Acceptance Criteria:**
- `ANTHROPIC_API_KEY` — auto-detected, registers Anthropic provider
- `OPENAI_API_KEY` — auto-detected, registers OpenAI provider
- `GOOGLE_API_KEY` — auto-detected, registers Google provider
- Only providers with valid keys are registered
- No prefix required — uses the same env var names as the native SDKs
