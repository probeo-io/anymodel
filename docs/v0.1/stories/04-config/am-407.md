## AM-407: Custom OpenAI-compatible provider config
**As a** developer
**I want** to add custom OpenAI-compatible endpoints via config
**So that** I can use Groq, Together, Ollama, vLLM, or any other compatible service

**Acceptance Criteria:**
- Configured under `providers.custom.{name}`
- Requires `baseURL`, optional `apiKey`
- Optional `models` array to manually list available models
- Auto-discovers models via `GET /models` on the endpoint if `models` not specified
- Referenced as `{name}/model-id` in requests
