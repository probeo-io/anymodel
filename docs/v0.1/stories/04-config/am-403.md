## AM-403: JSON config file
**As a** developer
**I want** to use an `anymodel.config.json` file for full configuration
**So that** I can define providers, aliases, defaults, and routing in one place

**Acceptance Criteria:**
- Looks for `anymodel.config.json` in working directory
- Also accepts `anymodel.config.js` (for dynamic config / env var interpolation)
- Supports `${ENV_VAR}` interpolation in string values
- Sections: `providers`, `aliases`, `defaults`, `routing`, `batch`
