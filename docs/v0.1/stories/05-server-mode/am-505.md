## AM-505: Batch endpoints
**As a** client application
**I want** batch CRUD endpoints served by the server
**So that** I can manage batches over HTTP

**Acceptance Criteria:**
- `POST /api/v1/batches` — create batch
- `GET /api/v1/batches` — list batches
- `GET /api/v1/batches/:id` — batch status
- `GET /api/v1/batches/:id/results` — batch results
- `DELETE /api/v1/batches/:id` — cancel batch
- Server manages polling and persistence internally
- Webhook delivery for completed batches
