## AM-510: Summary logging
**As an** operator
**I want** each request logged with model, tokens, cost, and latency
**So that** I can monitor usage without overwhelming log volume

**Acceptance Criteria:**
- Default log level: `summary`
- Each request logs: timestamp, model, provider, prompt tokens, completion tokens, estimated cost, latency ms
- Format: structured JSON (one line per request)
- Errors logged with error code and message
