## AM-504: Models endpoint
**As a** client application
**I want** `GET /api/v1/models` to return available models
**So that** I can discover what models the server offers

**Acceptance Criteria:**
- Returns all models from all configured providers
- OpenRouter-compatible format (id, name, pricing, context_length, etc.)
- Cacheable (models don't change during server lifetime unless refreshed)
